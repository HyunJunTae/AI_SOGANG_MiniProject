# FNN NSMC 모델 XAI 적용 및 결과 분석

## 1. 분석 대상

이 노트북은 NSMC 영화 리뷰 감성분석을 수행하는 FNN 모델에 대해 XAI 기법을 적용한다.

분석에 사용한 문장은 다음과 같다.

```text
이 영화 진짜 시간 가는 줄 모르고 봤네요 추천합니다
```

모델은 이 문장을 `긍정` 클래스로 예측했으며, XAI 파트에서는 해당 예측에 어떤 단어들이 영향을 주었는지 확인한다.

## 2. XAI 적용 전 준비 과정

XAI 함수들은 학습된 FNN 모델과 학습 과정에서 만든 전처리 자원을 그대로 사용한다.

사용되는 주요 객체는 다음과 같다.

```python
CLASS_NAMES = ["부정", "긍정"]
PAD_ID = word_to_index["<PAD>"]
UNK_ID = word_to_index["<UNK>"]
```

`CLASS_NAMES`는 모델 출력 인덱스 `0`, `1`을 각각 `부정`, `긍정`으로 해석하기 위한 이름표이다.

`PAD_ID`는 고정 길이 입력을 만들기 위해 빈 자리를 채우는 토큰 번호이고, `UNK_ID`는 학습 vocabulary에 없는 단어를 대체하는 토큰 번호이다.

새 문장은 `preprocess_text_for_model()` 함수를 통해 학습 때와 같은 방식으로 처리된다.

```text
문장 입력
-> clean_text로 정제
-> Okt 형태소 분석
-> 불용어 제거
-> word_to_index로 숫자 인코딩
-> MAX_LEN 길이로 패딩
-> FNN 모델 입력 생성
```

이 과정을 거쳐 문자열 문장이 FNN 모델이 받을 수 있는 정수 배열로 변환된다.

## 3. 적용된 XAI 기법

### 3.1 LIME

LIME은 특정 샘플 주변에서 모델의 동작을 단순한 설명 모델로 근사하는 기법이다.

노트북에서는 `explain_with_lime()` 함수로 적용했다.

```python
lime_exp = explain_with_lime(sample_text)
```

적용 순서는 다음과 같다.

```text
1. 원문 문장을 FNN 모델에 넣어 예측 확률 계산
2. 예측 확률이 가장 높은 클래스를 target_label로 선택
3. 원문에서 일부 단어를 제거한 변형 문장들을 생성
4. 변형 문장들을 FNN 모델에 넣어 예측 확률 수집
5. 그 결과를 바탕으로 설명용 선형 모델 학습
6. 단어별 weight를 계산하여 표와 그래프로 출력
```

여기서 `num_samples=500`은 LIME이 내부적으로 만들어 평가하는 변형 문장 수이고, `num_features=10`은 최종적으로 보여줄 중요 단어 수이다.

LIME 결과에서 weight는 현재 예측 클래스인 `긍정`에 대한 기여도를 의미한다.

```text
weight > 0: 긍정 예측을 강화한 단어
weight < 0: 긍정 예측을 약화한 단어
```

### 3.2 Occlusion

Occlusion은 단어를 하나씩 가려 보면서 모델 예측 확률이 얼마나 변하는지 직접 측정하는 기법이다.

노트북에서는 `explain_with_occlusion()` 함수로 적용했다.

```python
occlusion_result = explain_with_occlusion(sample_text)
```

적용 순서는 다음과 같다.

```text
1. 원문 문장을 전처리하여 토큰과 모델 입력 생성
2. 원문에 대한 긍정 확률 계산
3. 각 토큰을 하나씩 PAD_ID로 대체한 occluded 문장 생성
4. occluded 문장들을 다시 모델에 입력
5. 원문 확률과 occlusion 후 확률의 차이를 importance로 계산
6. 중요도가 큰 단어를 표와 그래프로 출력
```

중요도 계산은 다음과 같다.

```text
importance = 원문 긍정 확률 - 해당 단어를 가렸을 때의 긍정 확률
```

따라서 importance가 양수이면 해당 단어가 긍정 예측에 기여했다는 뜻이고, 음수이면 해당 단어를 가렸을 때 오히려 긍정 확률이 올라갔다는 뜻이다.

### 3.3 SHAP

노트북에는 SHAP 함수도 정의되어 있지만, 실제 실행 코드는 주석 처리되어 있다.

```python
# shap_values = explain_with_shap(sample_text)
```

따라서 현재 결과 분석은 실제 실행된 LIME과 Occlusion을 중심으로 해석한다.

## 4. LIME 결과 분석

LIME 결과에서 가장 큰 긍정 기여 단어는 `추천합니다`이다.

| token | weight | 해석 |
|---|---:|---|
| 추천합니다 | 0.234162 | 긍정 예측에 가장 크게 기여 |
| 진짜 | 0.079587 | 긍정 방향으로 기여 |
| 가는 | 0.050796 | 긍정 방향으로 약하게 기여 |
| 봤네요 | 0.025270 | 긍정 방향으로 약하게 기여 |
| 시간 | -0.143122 | 긍정 예측을 약화 |
| 영화 | -0.129910 | 긍정 예측을 약화 |
| 줄 | -0.110901 | 긍정 예측을 약화 |
| 이 | -0.045334 | 긍정 예측을 약하게 약화 |
| 모르고 | -0.028234 | 긍정 예측을 약하게 약화 |

LIME 기준으로 모델은 `추천합니다`를 매우 강한 긍정 단서로 사용했다. `진짜`, `가는`, `봤네요`도 긍정 쪽으로 작용했지만 영향력은 상대적으로 작다.

반면 `시간`, `영화`, `줄`은 음수 weight를 가진다. 이는 해당 단어들이 이 문장에서 긍정 예측에 직접적인 근거로 쓰이기보다는, LIME이 만든 주변 변형 문장들 안에서 긍정 클래스와 반대 방향의 패턴으로 해석되었음을 의미한다.

## 5. Occlusion 결과 분석

Occlusion 결과에서도 가장 중요한 단어는 `추천`으로 나타났다.

| token | importance | occluded_score | 해석 |
|---|---:|---:|---|
| 추천 | 0.443080 | 0.283691 | 해당 단어를 가리면 긍정 확률이 크게 하락 |
| 가다 | 0.100808 | 0.625964 | 긍정 예측에 기여 |
| 보다 | 0.082774 | 0.643997 | 긍정 예측에 기여 |
| 모르다 | 0.069746 | 0.657026 | 긍정 예측에 기여 |
| 영화 | 0.059677 | 0.667095 | 긍정 예측에 약하게 기여 |
| 진짜 | 0.040641 | 0.686130 | 긍정 예측에 약하게 기여 |
| 줄 | -0.035875 | 0.762646 | 가리면 긍정 확률이 상승 |
| 시간 | -0.141298 | 0.868069 | 가리면 긍정 확률이 크게 상승 |

Occlusion 기준에서도 `추천`은 압도적으로 중요한 긍정 단서이다. 이 단어를 가렸을 때 긍정 확률이 크게 떨어졌으므로, 모델이 긍정 판단을 내리는 데 핵심적으로 사용한 단어라고 볼 수 있다.

`가다`, `보다`, `모르다`도 양수 importance를 보이므로 문맥상 긍정 판단에 도움을 준다. 이는 원문 표현인 `시간 가는 줄 모르고 봤네요`가 긍정적인 감상 표현으로 작용했기 때문으로 해석할 수 있다.

반대로 `시간`과 `줄`은 음수 importance를 가진다. 특히 `시간`을 가렸을 때 긍정 확률이 더 높아졌으므로, 모델은 `시간`이라는 단어 자체를 긍정 단서로 보지는 않았다고 해석할 수 있다.

## 6. LIME과 Occlusion 비교

두 기법 모두 `추천/추천합니다`를 가장 강한 긍정 근거로 판단했다. 따라서 이 모델이 해당 문장을 긍정으로 예측한 핵심 이유는 추천 표현이라고 볼 수 있다.

공통적으로 긍정에 기여한 단어는 다음과 같다.

```text
추천 또는 추천합니다
진짜
보았다 계열 표현
```

공통적으로 긍정 기여가 약하거나 음수로 나타난 단어는 다음과 같다.

```text
시간
줄
```

두 기법의 차이도 존재한다. LIME은 여러 변형 문장을 만든 뒤 설명용 선형 모델을 학습하므로 주변 샘플들의 조합 패턴을 반영한다. 반면 Occlusion은 단어 하나를 직접 제거했을 때의 예측 확률 차이를 계산한다.

이 때문에 `영화`처럼 LIME에서는 음수로 나타났지만 Occlusion에서는 양수로 나타나는 단어가 있을 수 있다. 이는 두 기법이 중요도를 계산하는 방식이 다르기 때문이다.

## 7. 최종 해석

FNN 모델은 분석 문장을 `긍정`으로 판단할 때 `추천/추천합니다`를 가장 중요한 근거로 사용했다.

또한 `진짜`, `가다`, `보다`, `모르다`와 같은 단어들이 문맥상 긍정적인 감상 표현을 구성하면서 긍정 예측을 보조했다.

반면 `시간`, `줄`은 단독으로는 긍정 감성을 나타내는 단어가 아니기 때문에 일부 XAI 결과에서 부정적인 기여 또는 긍정 약화 요소로 나타났다. 그러나 문장 전체에서는 `시간 가는 줄 모르고`라는 표현이 긍정적인 관용적 의미를 만들고, 여기에 `추천합니다`가 결합되면서 최종적으로 강한 긍정 예측이 나온 것으로 해석할 수 있다.

따라서 이 XAI 결과는 모델이 감성 단어뿐 아니라 일부 문맥 표현도 반영하지만, 가장 명확한 감성 표현인 `추천`에 크게 의존하고 있음을 보여준다.
